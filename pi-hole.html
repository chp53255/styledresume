Network-Wide DNS Sinkhole (Pi-hole & Unbound)

Project write-up

Equipment/Software Used

  • Raspberry Pi 4B
  • Pi-hole
  • Raspberry Pi Zero 2W (for backup DNS)
  • Nebula-Sync (for syncing two Raspberry Pi devices)
  • CAT 6 Ethernet Cable + Crimping Tools

Project Description

This project involved setting up a Raspberry Pi 4B as a dedicated network security device. I installed Pi-hole to act as a DNS sinkhole, blocking advertisements and tracking domains at the network level for all connected devices. To increase privacy, I configured Unbound as a local, recursive DNS resolver, ensuring that DNS queries are resolved directly through root servers rather than third-party providers. I set up the same service on another Raspberry Pi Zero 2W device to serve as a backup DNS if the main Raspberry Pi 4B ever goes down. Using Nebula-Sync, I keep both devices in sync so changes on one Pi reflect on the other.

Skills Learned

  • Linux Administration: Installing and managing services via terminal.
  • Networking: Configuring static IP addresses and DHCP reservations.
  • DNS Security: Understanding the recursive resolution process and DNSSEC.
  • Privacy Hardening: Reducing the digital footprint of a local network.
